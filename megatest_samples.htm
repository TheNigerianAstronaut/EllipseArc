Introduction

This page contains EllipseArc examples that were generated using the megatest.php utility, which draws various kinds of shapes, some of which are arcs.
I wrote megatest.php to help me perform multiple tests on EllipseArc shapes.
Megatest.php is NOT part of the EllipseArc package: it's an entirely separate utility. It is technically not supported.

Megatest Usage

Megatest.php is executed from the command line. It reads a text file as input and produces an image which can be piped into an output image file: 
php megatest.php "megatest_input_text_file" > "output_image_file"
If executed without any arguments, it scans the current directory for any text files with a "megatest_input_file" prefix followed by two numeric digits. It will attempt to generate images for all these files.

Megatest Arguments

[BGIMGDIMS|CANVAS]=<img_type>=<img_width>=<img_height>=<colour>=<pct_transparency>=<rotation_degrees>=<shrink_width>=<shrink_height> // For the background image

IMAGEBORDER=<colour>=<pct_transparency>=<thickness>;

MOVE=<shape_id>,<delta_x>,<delta_y>

MOVEGROUP=<group_id_prefix>,<delta_x>,<delta_y>

MOVEALL=<delta_x>,<delta_y>

IMAGEFILE=<output_image_file_name>

  • The "|" character marks the end of a line entry
  • Use "//" to comment out a single line entry
  • Use the "|/*" and "*/|" pair to comment out a block of entries.
  • The "img_type" can be set to gif, jpg, png
  • The "shape_id" label does not have to be unique: each line entry is processed independently of the others so the exact same label can be used on different shapes. However using unique labels makes it easier for you (the human... or alien) to track what your shapes are doing. It is also useful if you want to move a shape or group of shapes.
  • A positive "thickness" will thicken the lines of the object where relevant
  • A zero "thickness" will result in a solidly filled object where relevant
  • A negative "thickness" will result in a hollow object where relevant
  • "BGIMGDIMS" defines the background image dimensions ie the canvas
  • "CANVAS" is an alias for BGIMGDIMS
  • "IMAGEBORDER" will draw a border of the specified colour around the image. It is the last thing drawn, so it will be on top of anything else that passes the border of the image.
  • "IMAGEFILE" saves the image to an output file with the name provided to this line.
  • "TRIM" causes the arc, circle or ellipse to remove the top and bottom tips which appear as pixels. The resulting shape looks smoother, however it may also make the final shape look blocky
  • "SMOOTH" targets only pixels that stick out. It is a softer option than TRIM. Try SMOOTH before going for TRIM.
  • "MOVE" will move the shapes whose names exactly match <shape_id>
  • "MOVEGROUP" will move any shapes whose names contain the <group_id_prefix> string
  • "MOVEALL" will move all shapes in the canvas
  • "LINEA" is an alias for "LINEANGLE"
  • "LINEG" is an alias for "LINEGRAD"
  • "LINEGRADIENT" is an alias for "LINEGRAD" (rather long, innit?)
  • "ARCC" is an alias for "ARCCHORD"
  • "CHORD" is an alias for "ARCCHORD"
  • "PIE" is an alias for "ARCSEGMENT"
  • "ARCS" is an alias for "ARCSEGMENT"
  • "SEGMENT" is an alias for "ARCSEGMENT"
  • "ARCT" is an alias for "ARCTRUECHORD"
  • "TCHORD" is an alias for "ARCTRUECHORD"
  • "RECT" is an alias for "RECTANGLE"
  • "TEXT" entries permit you to use the newline character \n or you can press the ENTER key directly
  • "width_thickness/height_thickness" specify the internal thickness of the width and height
  • "If the internal thickness of an arc exceeds the width or height, the thickness defaults to zero - so the shape will be solidly filled.
  • "FLIPV" will flip the shape vertically. An even number of FLIPV entries on the same shape will cancel out.
  • "FLIPH" will flip the shape horizontally. An even number of FLIPH entries on the same shape will cancel out.
  • "COMP" will draw the complement (ie opposite side) of the shape. An even number of COMP entries on the same shape will cancel out.
